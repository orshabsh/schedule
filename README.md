# School Schedule

Этот репозиторий содержит простой сайт расписания для учащихся и панель администратора для редактирования расписания с сохранением через GitHub API.

## Файлы

- `index.html` – страница для просмотра расписания учащимися.
- `admin.html` – визуальный редактор для администратора с авторизацией и сохранением в `schedule.json`.
- `schedule.json` – файл с данными расписания (хранится в корне репозитория).

## Как получить GitHub токен

1. Войдите в свой аккаунт на GitHub.
2. Перейдите в **Settings** (Настройки).
3. В меню слева выберите **Developer settings** → **Personal access tokens**.
4. Нажмите **Generate new token (classic)**.
5. В поле **Note** укажите любое имя (например, "Schedule Editor").
6. Установите **Expiration** (например, No expiration) по желанию.
7. В разделе **Scopes** отметьте:
   - Для публичных репозиториев: `public_repo`
   - Для приватных репозиториев: `repo`
8. Нажмите **Generate token** и скопируйте полученный токен **один раз**.

## Настройка репозитория на GitHub Pages

1. Создайте репозиторий (например, `schedule`).
2. Загрузите файлы `index.html`, `admin.html`, `schedule.json` и `README.md` в корень репозитория.
3. В настройках репозитория зайдите в **Settings** → **Pages**.
4. В разделе **Source** выберите ветку `main` и папку `/ (root)`, нажмите **Save**.
5. После этого страницы будут доступны по ссылкам:
   - `https://<ваш-ник>.github.io/schedule/index.html`
   - `https://<ваш-ник>.github.io/schedule/admin.html`

## Использование панели администратора

1. Откройте `admin.html` (через GitHub Pages).
2. При первом входе введите пароль администратора: **admin1977**.
3. В панели:
   - Выберите класс и день недели.
   - Вносите изменения в таблицу (8 уроков).
   - При необходимости добавьте или удалите класс.
4. Нажмите **Сохранить на GitHub**.
5. При первом сохранении вам потребуется заменить в коде `admin.html` placeholder `"ТОКЕН"` на ваш персональный токен GitHub.
6. Права для сохранения:
   ```javascript
   const token = "ВАШ_ТОКЕН";
   ```
7. После успешного сохранения данные обновятся в `schedule.json` и станут доступны на странице `index.html`.
